## Summary

Adds automated Proxmox deployment workflow with Taskfile tasks and shell scripts for repeatable, safe container management.

## Changes

### Shell Scripts (`scripts/proxmox/`)

- **`create-container.sh`**: Creates privileged LXC container with Docker pre-installed
- **`deploy.sh`**: Full deployment pipeline (tarball → transfer → build → migrate)
- **`status.sh`**: Shows container status, IP, and access URLs
- **`logs.sh`**: Streams application logs

### Taskfile Tasks

- `task proxmox:create-container` - Create new container (with safety checks)
- `task proxmox:deploy` - Deploy or update application
- `task proxmox:status` - View deployment status
- `task proxmox:logs [SERVICE=backend]` - Stream logs
- `task proxmox:update` - Alias for deploy

### Safety Features

- ✅ Checks if container ID already exists before creating
- ✅ Validates Proxmox connection
- ✅ Auto-generates secure credentials (database password, SECRET_KEY)
- ✅ Reads `ALPHA_VANTAGE_API_KEY` from local `.env`
- ✅ Proper error handling with colored output
- ✅ Idempotent operations (safe to run multiple times)

### Documentation

- `scripts/proxmox/README.md` - Comprehensive usage guide
- Updated `docs/deployment/proxmox-deployment.md` with manual steps reference

## Environment Variables

All tasks support customization:
- `PROXMOX_HOST` (default: `root@proxmox`)
- `PROXMOX_CONTAINER_ID` (default: `107`)
- `PROXMOX_CONTAINER_HOSTNAME`, `_MEMORY`, `_CORES`, `_DISK`

## Example Usage

```bash
# Create container (one-time)
task proxmox:create-container

# Deploy application
task proxmox:deploy

# Check status
task proxmox:status

# Update after code changes
git pull origin main
task proxmox:update
```
